# AI/ML Security Threats

## Platform

TryHackMe

## Objective

Understand common security threats that affect Artificial Intelligence (AI) and Machine Learning (ML) systems.

## Topics Covered

- AI Security Fundamentals
- Machine Learning Risks
- Adversarial Attacks
- Data Poisoning
- Model Theft
- Prompt Injection
- Privacy Risks

## Key Learning Points

### Adversarial Attacks

Attackers can manipulate input data to trick machine learning models into producing incorrect results.

### Data Poisoning

Malicious data can be injected into training datasets, causing the model to learn incorrect behavior.

### Model Theft

Attackers may attempt to copy or extract a trained model through repeated queries and analysis.

### Prompt Injection

Large Language Models (LLMs) can be manipulated through crafted prompts that bypass intended instructions.

### Privacy Risks

Sensitive information may be exposed through model outputs, training data leakage, or poor access controls.

## Security Recommendations

- Validate training data sources.
- Monitor model behavior for anomalies.
- Implement access controls for models and APIs.
- Protect sensitive training data.
- Test AI systems against adversarial inputs.

## Lessons Learned

This room introduced the security challenges associated with AI and Machine Learning systems and highlighted the importance of securing data, models, and AI-powered applications.
